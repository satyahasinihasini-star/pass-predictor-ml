# Pass Predictor — Mini Project (Frontend + ML + DBMS)

A mini web app that predicts whether a student will **PASS** or **FAIL** based on study hours, attendance, and previous marks — combining a custom frontend, a trained ML model, and a database.

## Tech Stack

- **Frontend:** HTML, CSS, JavaScript — custom slider controls + animated SVG gauge (no frameworks)
- **ML:** Python, scikit-learn (Decision Tree Classifier)
- **Backend:** Python, Flask
- **DBMS:** SQLite — every prediction is logged with a timestamp

## How It Works

1. You drag three sliders: study hours/day, attendance %, previous marks %
2. Clicking **Run Prediction** sends those values to the Flask `/predict` route
3. Flask loads the trained model (`model.pkl`) and predicts PASS/FAIL + a probability
4. The result is saved into `database.db` (SQLite) and shown on an animated gauge
5. The **Recent runs** panel pulls the last 10 predictions from the database via `/history`

## How to Run

1. Install dependencies:
   ```
   pip install -r requirements.txt
   ```

2. Train the model (only needed once, or whenever you want to retrain):
   ```
   python model_train.py
   ```
   This creates `model.pkl`.

3. Start the server:
   ```
   python app.py
   ```

4. Open your browser at:
   ```
   http://127.0.0.1:5000
   ```

The database file (`database.db`) is created automatically the first time you run `app.py`.

## Notes for the Demo / Viva

- The dataset is synthetically generated in `model_train.py` (400 samples) using a weighted rule + random noise, to simulate realistic student data.
- Model used: **Decision Tree Classifier** (`max_depth=4`), test accuracy ~85-90% depending on random seed.
- You can swap in a real dataset (e.g., from Kaggle) by replacing the data-generation code in `model_train.py` with `pd.read_csv(...)`, keeping the same three feature columns.
- The gauge needle and color (red → amber → teal) reflect the model's predicted probability, not just a binary PASS/FAIL.
